# SSAS Version Features

## SSAS 2019 (SQL Server 2019)

Compatibility level 1500. Released November 2019.

### Calculation Groups
- Address the problem of measure proliferation in complex BI models
- Common calculations (time intelligence patterns like YTD, QTD, YoY%) are defined once as calculation items rather than duplicated across every base measure
- Support custom ordering of calculation items
- Limitation: Row-level security (RLS) cannot be applied to calculation groups directly or indirectly

### Many-to-Many Relationships
- Native support for many-to-many relationships without bridge tables in DAX
- Requires compatibility level 1500+
- Can be created via Visual Studio 2019 (VSIX 2.9.2+), Tabular Object Model (TOM) API, TMSL, or Tabular Editor
- Simplifies modeling patterns that previously required complex DAX workarounds

### Query Interleaving
- Short query bias for high-concurrency workloads
- Prior behavior: first-in-first-out (FIFO) CPU scheduling meant fast queries could be blocked behind slow queries
- Query interleaving allows concurrent queries to share CPU resources so fast queries are not starved
- Configurable via system settings for tuning concurrency behavior

### Governance Settings
- **ClientCacheRefreshPolicy**: Controls how client-side query caches are refreshed
- Originally introduced for Azure Analysis Services, now available in on-prem SSAS 2019
- Allows administrators to manage when cached data is invalidated after model processing

### Other 2019 Features
- Superimposed relationships
- Online attach for read-only workloads
- Power BI dataset connectivity improvements

## SSAS 2022 (SQL Server 2022)

Compatibility level 1600. Released November 2022.

### DirectQuery Improvements
- **Parallel DirectQuery**: The engine identifies independent storage engine operations within a DAX query and executes them concurrently against the source database
- **MaxParallelism property**: Controls the number of parallel threads to prevent overburdening the data source
- Significant performance improvement for DirectQuery-heavy workloads

### Composite Models / Power BI Integration
- Power BI models can now make DirectQuery connections to SSAS 2022 Tabular models
- Enables data modelers to combine imported data in Power BI with live-connected SSAS data
- Requires Power BI Desktop May 2022 or later

### Object-Level Security (OLS)
- Table-level and column-level security defined within roles
- Restricts visibility of entire tables or columns for specific roles
- Members of roles with OLS cannot see restricted objects in any tool or query
- Complements existing row-level security

### MDX Fusion
- Formula Engine optimization that reduces the number of Storage Engine queries per MDX query
- Originally developed for Power BI, now brought to on-prem SSAS
- Improves MDX query performance without requiring query rewrites

### Dynamic Management Views (DMVs)
- Improved DMV support for better diagnostics and monitoring
- Enhanced metadata discovery capabilities

### Other 2022 Features
- Compatibility level 1600
- Improved error messages and diagnostics
- Updated client library requirements

## SSAS 2025 (SQL Server 2025)

Compatibility level 1700 (expected). Currently in preview/GA phase.

### Performance Enhancements

**MDX + Calculation Groups Performance:**
- Significant performance improvements for MDX queries on models with Calculation Groups and Format Strings
- Reduced memory usage and improved responsiveness for these specific query patterns

**Enhanced DirectQuery Parallelism:**
- Builds on SSAS 2022 parallel DirectQuery
- Parallelizes multiple queries to the data source for a single DAX query
- Reduces impact of data source delays and network latencies

**Horizontal Fusion (Updated):**
- Latest version of the Horizontal Fusion optimization
- Reduces the number of SQL queries generated by DAX in DirectQuery mode
- Fewer round-trips to the source database means faster overall query execution

### Modeling and Calculation Improvements

**Selection Expressions:**
- Fine-grained control over calculation behavior when multiple calculation items are selected
- Additional logic for handling cases where no specific selection is made on a calculation group
- Extends the calculation groups feature introduced in SSAS 2019

**Independent Value Filters:**
- New option to control value filter behavior
- Prevents automatic combining (coalescing) of multiple filters on the same table into a single filter
- Provides greater flexibility for precise, independent filtering in complex models

### DAX Language Enhancements

**LINEST and LINESTX Functions:**
- Perform linear regression using the Least Squares method
- Calculate the straight line that best fits given data points
- Enables statistical analysis directly within DAX without external tools

**Unicode Surrogate Pair Support:**
- Support for expanded Unicode standards including Chinese GB18030 government standard
- Ensures correct handling of supplementary Unicode characters in DAX string operations

### Communication and Performance

**Binary XML + Compression:**
- XMLA-based communication switched from plain text XML to binary XML
- Compression enabled for .NET client libraries
- Reduces network overhead and improves response times for large result sets

### Diagnostics Improvements

**Execution Metrics via XEvents:**
- Query execution metrics now exposed through Extended Events and Profiler traces
- Enables detailed performance analysis and query-level diagnostics
- Better visibility into query execution plans and resource consumption

## Migration: Multidimensional to Tabular

### When to Migrate
- Moving to Azure or cloud infrastructure (AAS and Fabric are Tabular-only)
- Adopting Power BI as the primary reporting front end
- Planning to use Microsoft Fabric
- Need for real-time data via DirectQuery
- Wanting modern tooling (Tabular Editor, DAX Studio, ALM Toolkit)

### When NOT to Migrate
- Existing solution is stable and users are satisfied
- Heavy reliance on MDX-only features (writeback, linked measure groups, actions)
- Significant investment in MDX-based reporting that would require complete rewriting
- No cloud migration planned

### Why Migrate
- All new Microsoft feature investment is in the Tabular engine
- Cloud services (AAS, Fabric) are Tabular-only
- Better compression and in-memory performance for most workloads
- Richer ecosystem of modern tools
- DAX is more accessible than MDX for new developers
- Power BI integration is native and seamless

### How to Migrate
1. **Assessment**: Inventory existing cubes, MDX queries, reports, and downstream consumers
2. **Redesign**: Migration requires a complete redesign (not a conversion). Rebuild the data model using tables/relationships instead of cubes/dimensions
3. **Rewrite queries**: Convert MDX to DAX (there is no automated converter)
4. **Retrain teams**: DAX is fundamentally different from MDX
5. **Parallel run**: Run both systems for 1-2 weeks, comparing query results and performance
6. **Staged cutover**: Migrate users in waves, not all at once
7. **Budget**: Expect 60-80% of original development time for the migration effort

## SSAS vs. Power BI Premium/Fabric Semantic Models

### Convergence Trend
Microsoft is consolidating analytical platforms toward Power BI/Fabric:
- Power BI Premium per-capacity SKUs are being retired in favor of Fabric capacity (F SKUs)
- Power BI in Fabric is positioned as a superset of Azure Analysis Services
- Same underlying VertiPaq engine across all platforms
- XMLA endpoint compatibility enables tool portability

### Key Differences

| Aspect | SSAS On-Prem | Azure AAS | Power BI / Fabric |
|--------|-------------|-----------|-------------------|
| Storage modes | Tabular + Multidimensional | Tabular only | Tabular only |
| Memory model | All models share server RAM | All models on single server | Per-model memory limits |
| Refresh | Manual/scheduled processing | Manual/scheduled processing | Better parallel refresh, incremental refresh built-in |
| Management | Self-managed | Azure PaaS | Fully managed SaaS |
| Cost model | SQL Server license | Azure SKU hourly | Fabric capacity subscription |
| Future investment | Maintenance mode | Migration recommended | Primary investment target |

### Strategic Recommendation
- **New projects**: Start with Power BI/Fabric semantic models unless specific on-prem requirements exist
- **Existing SSAS Multidimensional**: Evaluate migration to Tabular, then to Fabric
- **Existing SSAS Tabular**: Migrate to Fabric when ready; model compatibility is high
- **Existing Azure AAS**: Microsoft actively recommends migration to Power BI/Fabric
