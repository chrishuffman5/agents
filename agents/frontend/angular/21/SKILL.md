# Angular 21 Specialist

You are a specialist in Angular 21 (Active current, released November 2025). Angular 21 is the most structurally significant release since Angular 17 introduced signals. The defining change is that **Zone.js is no longer included in new projects by default**. Alongside this, Angular ships Signal Forms, replaces Karma with Vitest, adds a headless accessible component library, integrates with AI tooling via MCP, and configures Tailwind CSS out of the box.

## Zoneless by Default

New Angular 21 projects are generated without Zone.js. Change detection runs via the Angular signals runtime -- no monkey-patching of browser APIs.

Zone.js is still available for existing apps via `provideZoneChangeDetection()`. It is not removed from the framework, only from the default scaffold.

### New Project Structure

```typescript
// app.config.ts -- generated by ng new in Angular 21
import { ApplicationConfig } from '@angular/core';
import { provideRouter } from '@angular/router';
import { provideExperimentalZonelessChangeDetection } from '@angular/core';

export const appConfig: ApplicationConfig = {
  providers: [
    provideExperimentalZonelessChangeDetection(),
    provideRouter(routes),
  ],
};
```

```typescript
// main.ts -- no zone.js import
import { bootstrapApplication } from '@angular/platform-browser';
import { appConfig } from './app/app.config';
import { AppComponent } from './app/app.component';

bootstrapApplication(AppComponent, appConfig).catch(console.error);
```

### Writing Zoneless-Compatible Components

In zoneless mode, only signals trigger change detection. Imperative property mutations do not update the view.

```typescript
// WRONG -- plain property mutation does not trigger CD
@Component({ template: `{{ count }}` })
export class CounterComponent {
  count = 0;
  increment() { this.count++; } // Angular will NOT detect this
}

// CORRECT -- use signals
@Component({
  template: `
    <p>Count: {{ count() }}</p>
    <p>Double: {{ double() }}</p>
    <button (click)="increment()">+</button>
  `,
})
export class CounterComponent {
  count = signal(0);
  double = computed(() => this.count() * 2);
  increment() { this.count.update(n => n + 1); }
}
```

### Restoring Zone.js for Existing Apps

```typescript
// app.config.ts -- opt back into Zone.js
import { provideZoneChangeDetection } from '@angular/core';

export const appConfig: ApplicationConfig = {
  providers: [
    provideZoneChangeDetection({ eventCoalescing: true }),
    provideRouter(routes),
  ],
};
```

```typescript
// main.ts -- restore zone.js polyfill
import 'zone.js';
import { bootstrapApplication } from '@angular/platform-browser';
```

### Automated Migration Schematic

```bash
ng update @angular/core --migrate-only onpush_zoneless_migration
```

This schematic:
1. Converts `ChangeDetectionStrategy.Default` components to `OnPush`
2. Replaces manual `markForCheck()` with signal-compatible patterns where detectable
3. Flags imperative state mutations for manual review

### Change Detection Comparison

| Mechanism | Trigger | Granularity | Zone.js Required |
|---|---|---|---|
| Default (Zone.js) | Any async event | Full component tree | Yes |
| OnPush (Zone.js) | Input change or markForCheck | Subtree | Yes |
| Signals (zoneless) | Signal value change | Only consuming components | No |
| `markForCheck()` | Manual | Subtree | No |

## Signal Forms (Experimental)

Angular 21 introduces a third forms API built entirely on signals. Experimental -- API may change. Coexists with Reactive Forms and Template-driven Forms.

### Basic Usage

```typescript
import { formGroup, formField, Validators } from '@angular/forms/experimental';

@Component({
  template: `
    <form (ngSubmit)="submit()">
      <input [signalField]="form.fields.email" type="email" />
      @if (form.fields.email.errors()?.required) {
        <span>Email is required</span>
      }

      <input [signalField]="form.fields.password" type="password" />

      <button type="submit" [disabled]="!form.valid()">Login</button>
    </form>
  `,
})
export class LoginComponent {
  form = formGroup({
    email: formField('', { validators: [Validators.required, Validators.email] }),
    password: formField('', { validators: [Validators.required, Validators.minLength(8)] }),
  });

  submit() {
    if (this.form.valid()) {
      console.log(this.form.value()); // { email: string; password: string }
    }
  }
}
```

### Nested Forms

```typescript
const userForm = formGroup({
  name: formField('', { validators: [Validators.required] }),
  address: formGroup({
    street: formField('', { validators: [Validators.required] }),
    city: formField(''),
    zip: formField(''),
  }),
});

// Accessing nested values -- fully typed
console.log(userForm.value().address.city);
console.log(userForm.fields.address.valid()); // computed signal
```

### Repeating Groups (Arrays)

```typescript
import { formGroup, formField, formArray } from '@angular/forms/experimental';

form = formGroup({
  items: formArray([
    formGroup({
      name: formField(''),
      quantity: formField(1),
    }),
  ]),
});

addItem() {
  this.form.fields.items.push(
    formGroup({ name: formField(''), quantity: formField(1) })
  );
}
```

```html
@for (item of form.fields.items.controls(); track item) {
  <div>
    <input [signalField]="item.fields.name" />
    <input [signalField]="item.fields.quantity" type="number" />
  </div>
}
```

### Signal Forms vs Existing APIs

| Feature | Template-driven | Reactive Forms | Signal Forms |
|---|---|---|---|
| State location | Template directives | Class properties | Signals |
| Type safety | Limited | With typed forms | Full, automatic |
| Nested forms | Awkward | Manual wiring | First-class |
| Reactivity | Zone-driven | Observable streams | Signal graph |
| Testing | DOM-heavy | Unit testable | Unit testable |
| Status | Stable | Stable | Experimental |

## Vitest as Default Test Runner

Angular 21 replaces Karma with Vitest as the default unit test runner.

### Configuration

```json
// angular.json
{
  "test": {
    "builder": "@angular/build:unit-test",
    "options": {
      "buildTarget": "my-app:build",
      "runner": "vitest"
    }
  }
}
```

```bash
ng test          # Vitest in watch mode
ng test --once   # single run (CI)
```

### Writing Tests

```typescript
import { TestBed } from '@angular/core/testing';
import { CounterComponent } from './counter.component';

describe('CounterComponent', () => {
  beforeEach(() => {
    TestBed.configureTestingModule({
      imports: [CounterComponent],
    });
  });

  it('increments the count signal', () => {
    const fixture = TestBed.createComponent(CounterComponent);
    const component = fixture.componentInstance;
    expect(component.count()).toBe(0);
    component.increment();
    expect(component.count()).toBe(1);
  });
});
```

### Vitest Features

**Fake Timers:**

```typescript
import { vi } from 'vitest';

vi.useFakeTimers();
component.onInput('angular');
vi.advanceTimersByTime(300);
expect(component.searchTerm()).toBe('angular');
vi.useRealTimers();
```

**Import Mocking:**

```typescript
vi.mock('@angular/router', () => ({
  Router: vi.fn().mockImplementation(() => ({
    navigate: vi.fn(),
  })),
}));
```

**Snapshot Testing:**

```typescript
expect(fixture.nativeElement.innerHTML).toMatchSnapshot();
```

### Migrating from Karma


## @angular/aria (Developer Preview)

A headless accessible component library implementing WAI-ARIA design patterns. Provides behavior (keyboard navigation, ARIA attributes, focus management) but no styles.

### Installation

```bash
npm install @angular/aria
```

### Supported Patterns

| Pattern | Components |
|---|---|
| Listbox | `CdkListbox`, `CdkOption` |
| Combobox | `CdkCombobox`, `CdkComboboxPanel` |
| Dialog | `CdkDialog`, `CdkDialogContainer` |
| Menu | `CdkMenu`, `CdkMenuBar`, `CdkMenuItem`, `CdkMenuTrigger` |
| Accordion | `CdkAccordion`, `CdkAccordionItem` |
| Tree | `CdkTree`, `CdkTreeNode` |
| Tabs | `CdkTabs`, `CdkTabPanel` |
| Disclosure | `CdkDisclosure` |

### Example: Accessible Listbox

```typescript
import { CdkListbox, CdkOption } from '@angular/aria';

@Component({
  imports: [CdkListbox, CdkOption],
  template: `
    <ul cdkListbox [(value)]="selectedFruits" multiple aria-label="Select fruits">
      <li cdkOption="apple">Apple</li>
      <li cdkOption="banana">Banana</li>
      <li cdkOption="cherry">Cherry</li>
    </ul>
  `,
})
export class FruitPickerComponent {
  selectedFruits: string[] = [];
}
```

`CdkListbox` provides: `role="listbox"`, `role="option"`, arrow key navigation, Space/Enter selection, `aria-selected`, `aria-multiselectable`, and type-ahead.

### Example: Accessible Dialog

```typescript
import { CdkDialog } from '@angular/aria';

@Component({
  imports: [CdkDialog],
  template: `
    <button (click)="open()">Delete Item</button>
    <div cdkDialog [(open)]="isOpen" aria-labelledby="dialog-title">
      <h2 id="dialog-title">Confirm Deletion</h2>
      <p>Are you sure?</p>
      <button (click)="confirm()">Delete</button>
      <button cdkDialogClose>Cancel</button>
    </div>
  `,
})
export class ConfirmDialogComponent {
  isOpen = signal(false);
  open() { this.isOpen.set(true); }
  confirm() { this.isOpen.set(false); }
}
```

`CdkDialog` provides: `role="dialog"`, `aria-modal="true"`, focus trap, Escape to close, focus return, background scroll lock.

### @angular/aria vs Angular Material

| Concern | @angular/aria | Angular Material |
|---|---|---|
| Styles | None (headless) | Full Material Design |
| Customization | Unlimited | Theme system + overrides |
| Bundle size | Minimal | Larger |
| Use case | Custom design systems, Tailwind | Material Design apps |
| Status | Developer preview | Stable |

## HammerJS Removed

HammerJS touch gesture support is fully removed. `HammerModule`, `HammerGestureConfig`, and `HAMMER_GESTURE_CONFIG` no longer exist.

### Replacement: Native Pointer Events

```typescript
@Component({
  template: `<div (pointerdown)="onStart($event)" (pointerup)="onEnd($event)">Swipe me</div>`,
})
export class SwipeableComponent {
  private startX = 0;

  onStart(event: PointerEvent) { this.startX = event.clientX; }

  onEnd(event: PointerEvent) {
    const delta = event.clientX - this.startX;
    if (delta > 50) console.log('Swiped right');
    if (delta < -50) console.log('Swiped left');
  }
}
```

### Alternative Libraries

```bash
npm install @use-gesture/vanilla
# or
npm install interact.js
```

## Migration: v20 to v21

```bash
# Step 1: Run ng update
ng update @angular/core@21 @angular/cli@21

# Step 2: Address HammerJS removal
grep -r "HammerModule\|hammerjs\|HammerGestureConfig" ./src
npm uninstall hammerjs @types/hammerjs

# Step 3: Migrate to Vitest (optional but recommended)
ng update @angular/cli --migrate-only karma-to-vitest

# Step 4: Zoneless opt-in (optional for existing apps)
ng update @angular/core --migrate-only onpush_zoneless_migration

# Step 5: Update TestBed.get() calls
# Replace TestBed.get(Service) with TestBed.inject(Service)

# Step 6: Install @angular/aria (optional)
npm install @angular/aria
```

### Breaking Change Summary

| Change | Impact | Action Required |
|---|---|---|
| HammerJS removed | Apps using gestures | Replace with Pointer Events |
| `TestBed.get()` removed | Tests | Replace with `TestBed.inject()` |
| Zone.js default removed | New projects only | None for existing apps |
| Karma removed from scaffold | New projects only | None for existing apps |

## Pattern: Signal Forms with Validation

A complete example combining Signal Forms, zoneless change detection, and Tailwind CSS -- the natural combination in Angular 21.

```typescript
import { Component, inject, signal } from '@angular/core';
import { Router } from '@angular/router';
import { formGroup, formField, Validators } from '@angular/forms/experimental';

@Component({
  selector: 'app-registration',
  template: `
    <div class="mx-auto max-w-md space-y-6 p-8">
      <h1 class="text-2xl font-bold text-gray-900">Create Account</h1>

      <form (ngSubmit)="submit()" class="space-y-4">
        <div>
          <label class="block text-sm font-medium text-gray-700">Name</label>
          <input
            [signalField]="form.fields.name"
            type="text"
            class="mt-1 block w-full rounded-md border border-gray-300 px-3 py-2"
            [class.border-red-500]="form.fields.name.touched() && form.fields.name.invalid()"
          />
          @if (form.fields.name.touched() && form.fields.name.errors()?.required) {
            <p class="mt-1 text-sm text-red-600">Name is required.</p>
          }
        </div>

        <div>
          <label class="block text-sm font-medium text-gray-700">Email</label>
          <input
            [signalField]="form.fields.email"
            type="email"
            class="mt-1 block w-full rounded-md border border-gray-300 px-3 py-2"
          />
        </div>

        <button
          type="submit"
          [disabled]="form.invalid() || isSubmitting()"
          class="w-full rounded-md bg-blue-600 py-2 text-white font-medium
                 hover:bg-blue-700 disabled:opacity-50"
        >
          {{ isSubmitting() ? 'Creating...' : 'Create Account' }}
        </button>
      </form>
    </div>
  `,
})
export class RegistrationComponent {
  private auth = inject(AuthService);
  private router = inject(Router);

  form = formGroup({
    name: formField('', { validators: [Validators.required] }),
    email: formField('', { validators: [Validators.required, Validators.email] }),
    password: formField('', { validators: [Validators.required, Validators.minLength(8)] }),
  });

  isSubmitting = signal(false);
  serverError = signal<string | null>(null);

  async submit() {
    if (this.form.invalid()) return;
    this.isSubmitting.set(true);
    try {
      await this.auth.register(this.form.value());
      this.router.navigate(['/dashboard']);
    } catch {
      this.serverError.set('Registration failed.');
    } finally {
      this.isSubmitting.set(false);
    }
  }
}
```

Key patterns: signal form validation as computed signals, zoneless-compatible (all state is signals), Tailwind classes, `@if` control flow syntax.
